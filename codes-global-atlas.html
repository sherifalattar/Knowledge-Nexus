Anatomy

Code anatomy, without the fog machine

A practical reference architecture for reading codes faster, even when the exact code book differs between payers, TPAs, regulators, or EDI standards. Most systems use a compact syntax: alphabetic prefixes classify the family, numeric values identify the reason, and optional suffixes describe severity, routing, appealability, or document requirements.

PREFIX
Category / domain
AUTH for authorization, ELIG for eligibility, NCOV for non-covered service, DUPL for duplicate request, DOC for missing documentation.
000–999
Specific reason number
Narrows the issue: no prior approval, inactive policy, excluded benefit, coding mismatch, duplicate submission, or clinical insufficiency.
.A / -R
Optional modifier
A local layer that may indicate appeal path, resubmission requirement, documentation group, claim line, severity, or edit version.
Critical distinction

Global structure is not a global code book

There is no single denial-code dictionary used by every healthcare market. What is globally recognizable is the logic: short alphanumeric codes that compress a denial reason, responsible party, remittance context, or next action into a single readable identifier.

DUPL-002 Duplicate The request appears to duplicate a previously submitted or processed service. Cross-check request ID, member ID, service date, provider code, and existing authorization number.
Standards

Where formal standards fit

In formal remittance environments, denial and adjustment explanation often uses recognized code sets rather than local dashboard labels.

X12 · Remittance context

CARC + Group Code

Claim Adjustment Reason Code paired with a group code (e.g. contractual obligation, patient responsibility) explains why an adjusted amount differs from what was billed.

Additional explanation

RARC

Remittance Advice Remark Code supplements a CARC with extra context — what is missing, what is needed next, or which policy applied.

Implementation note — a local alphanumeric layer (AUTH-017.A) is the human-readable navigation surface; formal CARC/RARC/X12 codes are the standardized remittance layer. The two coexist: one for clarity, one for interoperability.
